# 项目审查记录

本文件记录当前项目的一次整体审查结论，便于后续接入本地数据集时有明确的检查顺序。

## 当前结构

- `dms/` 已按功能拆分为人脸检测、头姿估计、视线估计、疲劳检测、注意力判断、状态融合和可视化。
- `scripts/smoke_test.py` 可用于最小可行性测试。
- `scripts/run_dataset.py` 可用于批量处理本地视频数据集。
- `data/` 和 `outputs/` 已通过 `.gitignore` 避免提交大体积本地数据和生成结果。
- `docs/` 保存设计方案、项目结构说明和本审查记录。

## 已处理事项

- README 和数据目录说明已统一为中文。
- 代码中的关键业务逻辑已补充中文注释。
- 命令行参数说明和脚本输出已改为中文。
- `.gitignore` 已补充 `demo/`、`outputs/` 和本地数据集忽略规则。

## 需要重点关注的风险

- 头部姿态估计目前使用 6 点 `solvePnP` 和近似相机内参。该方法适合课程原型，但在摄像头角度、脸部遮挡或关键点抖动明显时会偏差较大。
- OpenCV 默认 `putText` 不支持中文字体，因此输出视频叠字仍保留英文标签，避免画面中出现乱码。
- 疲劳检测依赖视频前几秒的睁眼基线。正式测试时，视频开头应尽量保证驾驶员正常看前方且眼睛睁开。
- 本项目当前是规则版系统，阈值需要结合你们的数据集重新校准。

## 建议的数据集测试顺序

1. 先运行 `python scripts/smoke_test.py --download-face --keep-temp` 检查环境。
2. 将少量本地视频放入 `data/local/`，用 `--max-frames 100` 做快速检查。
3. 查看 `outputs/quick_check/summary.csv`，确认是否存在明显误判。
4. 对关键视频打开标注视频和 JSON，重点看 `yaw_delta`、`pitch_delta`、`ear`、`perclos_30s`。
5. 再运行完整批处理，并根据结果调整 `configs/default.yaml`。

## 后续改进方向

- 使用真实相机标定参数替代当前近似相机内参。
- 优先使用 MediaPipe 的人脸变换矩阵或更多稳定关键点来改进头姿估计。
- 在本地数据集上统计正常驾驶片段的头姿分布，再重新设定 yaw/pitch 阈值。
- 将规则输出作为基线，后续用随机森林、SVM 或时间序列模型学习最终状态。
